SAM Doc : Probes
This page last changed on Jul 02, 2012 by prodrigu.


This page provides information about the status of probes released by the SAM team. These probes are provided by different software teams.

The responsibility of some probes is being transfered to EGI/EMI projects. More information on this transition can be found in EGI and EMI pages.

Due to this transition period, the table below reports the plans for each existing probe:

  • probe to be maintained
  • probe to be discontinued

The SAM Team is also responsible for the provision of the following probe-related libraries:

  • python-Gridmon (for the old Konstantin's probes)
  • perl-TOM (logging stuff)
  • perl-GridMon (common libs used by perl probes, like checking proxy or running commands)

EGI Probes

  • grid-monitoring-org.ggus-probes
    • NodeType: sam-nagios, ops-monitor
    • Description: queries GGUS to check for open tickets for a SU or by site name that has been notified
  • grid-monitoring-org.nagiosexchange-probes
    • NodeType: sam-nagios, ops-monitor
    • Description: checks from Nagios Exchange
  • grid-monitoring-org.nrpe-probes
    • NodeType: none
    • Description: pushes NRPE configuration from a web server when the push is initation by NAGIOS over NRPE
  • grid-monitoring-probes-cadist
    • NodeType: sam-nagios
    • Description: checks IGTF CA certificates distribution version and consistency
  • grid-monitoring-probes-hr.srce
    • NodeType: sam-nagios
    • Description: CAdist, CertLifetime, DPM, DPNS, GRAM, GridFTP, GridProxy, MyProxy and VOMS probes |
  • grid-monitoring-probes-org.scas
    • NodeType: none
    • Description: checks SCAS configuration, certificate, processes, log file and connection |
  • grid-monitoring-probes-org.sam.sec
    • NodeType: sam-nagios
    • Description: several security probes executed from the WNs |
